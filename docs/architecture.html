terraform-gcp-cloud-sql Architecture Document

Cloud SQL Module

Executive Summary

A production-ready Terraform module for deploying managed relational databases on Google Cloud SQL. Supports PostgreSQL 15 and MySQL 8.0 with high availability (REGIONAL), independently-sized read replicas, Private Service Networking for private IP connectivity, automated backups with point-in-time recovery, configurable maintenance windows, Query Insights for performance monitoring, database flags for engine tuning, and multi-database/user provisioning within a single instance.

Component Breakdown

google_sql_database_instance.this

Primary Cloud SQL instance with configurable engine version, machine tier, SSD/HDD disk with autoresize, REGIONAL or ZONAL availability, SSL enforcement, authorized networks, backup configuration (schedule, retention, PITR), maintenance windows, Query Insights, and custom database flags.

google_sql_database.this

Multiple databases per instance with configurable charset and collation.

google_sql_user.this

Database users with password and type (BUILT_IN, CLOUD_IAM_USER, CLOUD_IAM_SERVICE_ACCOUNT).

google_sql_database_instance.read_replica

Read replicas inheriting the primary's engine version with independent tier, disk size, disk type, and availability type settings. Non-failover replicas for read scaling.

Private Service Networking

google_compute_global_address for VPC peering range allocation + google_service_networking_connection for Private Services Access to Cloud SQL.

Data Flow

  • Write Path: Applications connect to the primary instance via private IP (PSN) or public IP (authorized networks). Writes are processed by the primary and replicated to read replicas.
  • Read Path: Read-heavy workloads are distributed across read replicas using separate connection names, reducing primary load.
  • Backup: Automated backups run daily at configurable start time with 7-day retention. Transaction logs are retained for point-in-time recovery.
  • Failover: REGIONAL availability type provides automatic failover to a standby in a different zone within the same region.
  • Maintenance: Scheduled maintenance windows (day/hour) with canary or stable update tracks minimize disruption.

Industry Adaptation

  • Financial Services: REGIONAL HA for zero-downtime, SSL enforcement, private IP only, PITR for RPO compliance, read replicas for reporting workloads.
  • Healthcare: Private networking only (no public IP), IAM-based authentication, automated backups with extended retention for HIPAA compliance.
  • E-Commerce: Read replicas for catalog queries during peak traffic, autoresize disks for growing order databases, Query Insights for slow query optimization.
  • SaaS: Multi-database instances for tenant isolation, database flags for connection pooling, stable maintenance track for predictable updates.

Production Readiness Checklist

  • Set availability_type = "REGIONAL" for high availability
  • Enable deletion_protection = true (default)
  • Configure private IP with Private Service Networking
  • Disable public IP (enable_public_ip = false) in production
  • Enforce SSL connections (require_ssl = true)
  • Enable automated backups with PITR
  • Set backup retention to at least 7 days
  • Configure maintenance window during low-traffic hours
  • Enable Query Insights for performance monitoring
  • Deploy at least one read replica for read-heavy workloads
  • Enable disk autoresize with an appropriate limit
  • Use IAM-based authentication where possible

Deployment

module "cloud_sql" {
  source = "github.com/kogunlowo123/terraform-gcp-cloud-sql"

  project_id       = "my-project"
  instance_name    = "prod-postgres"
  database_version = "POSTGRES_15"
  region           = "us-central1"
  tier             = "db-custom-4-16384"
  availability_type = "REGIONAL"
  disk_size        = 100
  disk_type        = "PD_SSD"

  enable_private_ip = true
  enable_public_ip  = false
  private_network   = "projects/my-project/global/networks/default"
  require_ssl       = true

  backup_enabled                 = true
  point_in_time_recovery_enabled = true
  retained_backups               = 14

  databases = {
    "myapp"     = {}
    "analytics" = {}
  }

  users = {
    "appuser" = { password = "secure-password" }
  }

  read_replicas = {
    "replica-1" = { tier = "db-custom-2-8192" }
  }

  insights_config = {
    query_insights_enabled  = true
    record_application_tags = true
    record_client_address   = true
  }

  labels = { team = "backend", env = "production" }
}
